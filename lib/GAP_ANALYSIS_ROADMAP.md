# خارطة تحسين تحليل الفجوة (Gap Analysis)

## الرأي وترتيب التنفيذ

### 1️⃣ ترتيب المهارات المفقودة حسب الأولوية — **ننفّذ أولاً**
- **الوضع الحالي:** عندك `skillPriorityRanking` في `GapAnalysisResult` و `criticalSkills` في الـ Job. المهارات المفقودة حالياً تظهر بنفس ترتيب `requiredSkills` (يعني فيها أولوية ضمنية)، لكن غير معرّفة كـ "أولوية" في الـ UI.
- **المطلوب:** إرجاع/عرض **Missing Skills مرتبة تنازلياً حسب الأولوية** (أعلى أولوية أولاً)، مع دمج `criticalSkills` في حساب الأولوية إن وُجدت.
- **الفائدة:** المستخدم يعرف إيه أهم حاجة يشتغل عليها ويقدر يبني خطة تعلم واضحة.

---

### 2️⃣ تحليل الدورات المفقودة وربطها بالمهارات — **ننفّذ (نسخة خفيفة)**
- **الوضع الحالي:** عندك `requiredCourses` و `missingCourses` و `missingSkills` بدون ربط صريح (Course X → Skills Y).
- **الخيارات:**
  - **بدون DB كورسات:** نعمل mapping بسيط: لكل مهارة مفقودة نعرض "الدورات المطلوبة للوظيفة واللي أنت مش ماخدها" كتوصية عامة، مع إمكانية لاحقاً إضافة حقل في الـ Job مثل `courseToSkills` (Map أو قائمة أزواج).
  - **مع DB كورسات لاحقاً:** collection في Firestore يربط Skill ↔ Course فيقدر الـ App يقترح "عشان تتقن X خد الدورة Y".
- **التوصية:** نضيف في الـ Result بنية جاهزة مثل `Map<String, List<String>> skillToSuggestedCourses` (حتى لو قيمتها من الـ requiredCourses العامة دلوقتي)، ونعرض في الـ UI "لتطوير المهارة X ركّز على الدورات: ...". كده نجهّز للتوسع لاحقاً.

---

### 3️⃣ Visualization (Bar / Pie + Highlight للأولوية) — **ننفّذ**
- **الوضع الحالي:** النتائج نصوص ورقم Match %.
- **المطلوب:**
  - **Bar chart** (أو Pie): نسبة المهارات المطابقة vs المفقودة (وإن أمكن Technical vs Soft).
  - **Highlight** المهارات الأعلى أولوية في قائمة المفقودات (لون أو أيقونة "أولوية عالية").
- **الفائدة:** توضيح سريع وإبراز إيه اللي يستحق تركيز أول.

---

### 4️⃣ تحسين Match Percentage (وزن حسب الأولوية) — **ننفّذ**
- **الوضع الحالي:** 70% Technical + 30% Soft (نسبة مطابقة من العدد فقط).
- **المطلوب:** **Weighted match:** كل مهارة لها وزن (مثلاً من `skillPriorityRanking` أو من كونها critical). المعادلة: `weightedMatch = sum(weight_i × (1 if matched else 0)) / sum(weight_i)`.
- **الفائدة:** الـ Match يعكس "مدى تغطية أهم المهارات" مش بس عدد المهارات.

---

### 5️⃣ اقتراح كورسات ومهام تطبيقية — **نسخة أولى بدون DB**
- **بدون DB:** الـ App يقترح: (أ) قائمة الكورسات المطلوبة واللي مفقودة، (ب) "مشاريع تطبيقية مقترحة" كنص ثابت أو من وصف الوظيفة (مثلاً "اعمل مشروع تحليل بيانات حقيقي").
- **مع DB لاحقاً:** اقتراح كورسات مرتبطة بالمهارات + مهام/مسارات تعلم.

---

## ترتيب التنفيذ المقترح (تم تنفيذ 1–4)

| # | المهمة | الحالة |
|---|--------|--------|
| 1 | ترتيب Missing Skills بالأولوية + استخدام criticalSkills في الأولوية | تم: في `GapAnalysisService` + عرض مرتب في الـ UI |
| 2 | Weighted Match % حسب الأولوية | تم: `weightedMatchPercentage` والنسبة المعروضة في الهيدر والكارد |
| 3 | Visualization (Bar + Highlight أولوية) | تم: شريط أخضر/برتقالي + شارة "High priority" للمهارات الحرجة |
| 4 | Mapping مهارة → دورات مقترحة (نسخة خفيفة) | تم: `skillToSuggestedCourses` + عرض "Suggested courses" تحت كل مهارة مفقودة |
| 5 | اقتراح كورسات/مهام (نص + بنية لـ DB لاحقاً) | مؤجل: يمكن إضافة collection كورسات لاحقاً وربطها بالمهارات |

---

## ملاحظات تقنية

- **الأولوية:** يمكن تعريفها كالتالي: مهارات في `criticalSkills` تأخذ أعلى رتبة (مثلاً 1000+)، والباقي حسب ترتيبهم في `requiredSkills` (أول = أعلى).
- **الـ UI:** عرض المهارات المفقودة مرتبة تنازلياً مع شارة "أولوية عالية" للمهارات الحرجة، وربط كل مهارة مفقودة بسطر "دورات مقترحة" حتى لو من القائمة العامة دلوقتي.
